# 开发指南

## 1. 环境准备

1. Node.js 20+
2. pnpm 10+
3. Rust stable（桌面模式）

## 2. 安装依赖

```bash
pnpm install
```

## 3. 本地运行

```bash
pnpm dev
```

Tauri 桌面模式：

```bash
pnpm tauri dev
```

## 4. 常用命令

```bash
pnpm lint
pnpm build
pnpm preview
```

## 5. 当前阶段开发优先级

1. 先实现 Markdown MVP 主链路（实时编辑即预览）。
2. 再补保存、文件管理与基础测试。
3. 最后补安全策略、性能与 Rust 下沉。

## 6. 提交流程建议

1. 小步提交，每个提交可回滚。
2. 提交前至少运行 `lint + build`。
3. 大改动需要同时更新 `docs/*`。

## 7. 当前结构约束

1. 当前阶段业务代码优先放在 `App.tsx`。
2. `components/ui/*` 只放基础复用组件。
3. 自定义 hooks 统一放在 `src/hooks/*.ts`。
4. 类型定义统一放在 `src/types/*.d.ts`。
5. 暂不做 `features/*` 业务拆分，等业务稳定后再拆。
6. 组件命名保持简短，只有可复用时才抽成组件。
