# 组件迁移清单（旧组件 -> shadcn/ui）

## 1. 迁移原则

1. 不直接复刻旧样式，统一到 token 和 shadcn 组合。
2. 保留业务语义，替换视觉与交互外壳。
3. 每个组件迁移必须具备独立回滚点（按文件夹或功能区）。

## 4. 暂缓项策略

暂缓项仅允许以下两类：
1. 与 Tauri 原生窗口行为强耦合，且替换风险高。
2. 依赖 Monaco 低层 API，不适合在 P0/P1 同步重构。

每个暂缓项必须写明：
1. 暂缓原因
2. 风险评估
3. 计划解决阶段

## 5. 完成度统计口径

1. 组件数完成度：完成迁移组件数 / 清单总组件数。
2. 页面覆盖完成度：新 UI 覆盖交互路径数 / 总交互路径数。
3. 验收通过率：已迁移组件中通过单测+手测的比例。
