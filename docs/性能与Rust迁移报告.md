## 下沉候选清单（按优先级）

| 候选模块 | 预期输入/输出 | 预估收益 | 复杂度 | 优先级 |
| --- | --- | --- | --- | --- |
| TOC 提取 | Markdown -> 标题树 | 大文档目录解析提速 2~4x | 中 | 高 |
| 文档统计 | Markdown -> 字数/行数/段落数 | 高频计算 CPU 降低 30~50% | 低 | 高 |
| 工作目录索引 | 文件夹 -> 文档元数据列表 | IO + 过滤效率明显提升 | 中 | 中 |
| 搜索/过滤 | query + docs -> 匹配结果 | 大量文档时响应更稳 | 中 | 中 |

## 样例落地计划

样例 A：`extract_toc`（Rust）
1. 输入：Markdown 文本。
2. 输出：目录节点（level/title/slug）。
3. 前端用途：预览 TOC、快速跳转。

样例 B：`calc_doc_stats`（Rust）
1. 输入：Markdown 文本。
2. 输出：字符、单词、行数、代码块数量。
3. 前端用途：状态栏统计。

## 基准测试方法

1. 数据集：50KB、500KB、1MB Markdown。
2. 每组运行 50 次，记录 p50/p95/max。
3. 对比 TS 与 Rust 路径。

目标阈值：
1. 平均耗时下降 >= 40%。
2. 1MB 场景 p95 下降 >= 50%。

## 风险与回滚

1. 风险：跨语言接口复杂度增加。
2. 回滚：每个 Rust 能力保留 TS fallback，并以 `rust_accel` 开关切换。
