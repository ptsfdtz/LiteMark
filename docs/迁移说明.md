# 迁移说明

## 1. 适用范围

本文适用于从旧版 LiteMark 代码结构迁移到重构版架构（P0~P2）。

## 2. 迁移原则

1. 先加不改：先引入新结构，再替换旧实现。
2. 保留回滚：每个迁移点都有 feature flag。
3. 文档先行：迁移前写清输入、输出、风险、回滚。

## 3. 迁移步骤

1. 建立新目录骨架（`app/features/entities/shared`）。
2. 将公共逻辑迁移到 `shared`，保持旧入口可用。
3. 按功能区迁移组件到 shadcn/ui。
4. 切换默认入口到新实现。
5. 清理旧实现与兼容层。

## 4. 回滚步骤

1. 将 feature flag 退回旧路径：`ui_v2=false`、`markdown_pipeline_v2=false`、`rust_accel=false`。
2. 回滚到最近阶段 freeze tag。
3. 保留失败样例并补充测试。

## 5. 验收检查表

1. 编译通过：`pnpm build`
2. 代码规范：`pnpm lint`
3. 核心手测：打开/保存/预览/设置/最近文件
4. 文档同步：README、CONTRIBUTING、架构说明、变更日志
