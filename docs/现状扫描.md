# LiteMark 现状扫描（2026-03-05）

## 基线结论

1. 项目基线为 `React + Vite + Tailwind v4 + shadcn/ui + Tauri v2`。
2. Markdown MVP 已落地：编辑即预览（TipTap）、基础工具栏、导入/导出、草稿自动保存（localStorage）、字数/字符统计。
3. 桌面壳仍为基础形态：Tauri 无业务 command，文件系统能力尚未接入。
4. 安全与测试仍缺口：Markdown 渲染未做 sanitize，测试与性能基准未建立。

## 关键扫描证据

1. `src/App.tsx` 已实现编辑器主链路、导入/导出、自动保存与统计展示。
2. `src/lib/md.ts` 使用 `marked` + `turndown` 完成 Markdown/HTML 双向转换。
3. `src/hooks/useMarkdownStats.ts` 提供字数/字符统计。
4. `src-tauri/src/lib.rs` 仅包含基础 `run`，无业务命令。
5. `src-tauri/tauri.conf.json` 仍为默认窗口配置，`csp` 为 `null`。

## Top 10 问题

1. 文件管理缺口：无打开/保存/最近文件/工作目录模型（仅浏览器导入导出）。
2. 安全基线缺口：`marked` 渲染未接 sanitize 与协议白名单策略。
3. 桌面能力缺口：Tauri 未暴露文件与系统能力。
4. 测试缺口：无单元测试、集成测试、性能基准。
5. 性能边界未验证：大文档场景、转换成本与内存占用未量化。
6. 模式缺口：无预览/分屏/专注模式。
7. 设置缺口：无主题/字体/快捷键自定义。
8. 国际化缺口：全局文案仅 zh-CN。
9. 发布规范缺口：验收口径与回滚流程待完善。
10. 文档同步风险：文档需随功能演进持续更新。

## 已验证命令

1. `pnpm lint`：通过（2026-03-05）
2. `pnpm build`：通过（2026-03-05）

## 构建告警

1. Vite 提示 chunk size > 500k（当前打包为单大块）。

## 结论

当前处于“已具备 MVP 编辑链路，但桌面文件能力与安全/测试尚未补齐”的阶段。建议优先补文件能力与安全策略，再进入体验与性能深化。
