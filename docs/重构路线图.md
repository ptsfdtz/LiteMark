# LiteMark 重构路线图

> 版本：基于 2026-03-05 全新初始化代码扫描结果。

## 目标

1. 以渐进迁移方式完成 LiteMark 从“骨架”到“可发布版本”的重构落地。
2. 每个阶段具备：可验收、可回滚、可观测。
3. 严禁大爆炸重写。

## P0 / P1 / P2 阶段计划

工期预估（单主程 + 1 协作评审）：
1. P0：2 周（10 个工作日）
2. P1：3 周（15 个工作日）
3. P2：2 周（10 个工作日）

### P0（骨架补全）

改动清单：
1. 建立业务分层目录（`features`, `entities`, `shared`, `app`）。
2. 落地主界面与布局壳（编辑区、预览区、工具栏、侧栏）。
3. 完成 Markdown 渲染第一版（GFM + 代码块 + 表格 + 引用 + 任务列表）。
4. 建立 Tauri 文件命令（读/写/列举/重命名）。
5. 建立测试基座（Vitest）与首批用例。

风险与回滚：
1. 风险：基础架构调整导致目录震荡。
   回滚：每个功能区独立 PR，按模块回退。
2. 风险：Tauri 命令接入引入平台差异。
   回滚：先保留 mock fallback，在 dev 模式可切换。

测试清单：
1. 单测：Markdown 语法渲染至少 30 条样例。
2. 集成：编辑-预览联动。
3. 手测：打开/保存/另存为/重命名基础流程。

验收标准：
1. 有可用 MVP：能编辑并预览 Markdown，能保存本地文件。
2. `lint + build + test` 全通过。
3. Markdown 基础语法正确率 >= 95%。

### P1（体验与安全）

改动清单：
1. 完整模式行为（编辑态、预览态、分屏态）统一。
2. 完成安全策略（XSS 过滤、链接白名单、target/rel 规范）。
3. 完成 shadcn 业务组件组合（dialog/sheet/dropdown/tooltip 等）。
4. 增加设置、主题、国际化与最近文件体验。

风险与回滚：
1. 风险：模式切换状态复杂，容易出现交互回归。
   回滚：引入状态机与 feature flag，保留旧交互开关。
2. 风险：安全策略过严影响兼容。
   回滚：分级策略（strict/compat）可切换。

测试清单：
1. 单测：安全过滤与协议过滤用例。
2. 集成：三种模式切换与滚动同步。
3. 手测：大文档（>=1MB）编辑体验。

验收标准：
1. Markdown 核心渲染正确率 >= 99%。
2. 高危 XSS 用例 0 漏洞。
3. 关键交互路径无阻塞（主观延迟可接受）。

### P2（性能与 Rust 下沉）

改动清单：
1. 识别高频计算并下沉 Rust（至少 1~2 个样例）。
2. 建立性能基准脚本（前后对比）。
3. 优化打包策略（按需加载、懒加载）。
4. 完成发布文档与回滚预案。

风险与回滚：
1. 风险：Rust 接口演进导致调用不稳定。
   回滚：保留 TS fallback 并用开关控制。
2. 风险：跨平台构建复杂度上升。
   回滚：CI 保留纯前端链路，Rust 能力可选启用。

测试清单：
1. Rust 单测与基准测试。
2. 前后端集成测试。
3. 性能回归测试（首屏、渲染、滚动同步）。

验收标准：
1. 至少 1~2 个 Rust 样例上线。
2. 关键路径性能提升 >= 20%。
3. 构建与发布流程可复现。
