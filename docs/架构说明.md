# 架构说明

## 当前架构（初始化状态）

```text
src/
  App.tsx                # 入口页面（当前为空壳）
  main.tsx               # React 启动
  index.css              # Tailwind v4 + shadcn 设计 token
  components/
    ui/                  # shadcn 基础组件
    common/              # 业务组件预留目录
  lib/utils.ts           # cn 工具函数
src-tauri/
  src/lib.rs             # Tauri app 启动（无业务命令）
  tauri.conf.json        # 默认窗口与构建配置
```

## 目标架构（重构后）

```text
src/
  app/
    providers/
    styles/
  features/
    editor/
    preview/
    file-manager/
    settings/
  entities/
    document/
    preference/
  shared/
    ui/
    markdown/
    tauri/
    utils/
```

## 设计原则

1. UI 与业务分离：`components/ui` 不包含业务逻辑。
2. Feature 独立：每个 feature 内聚状态、服务、视图。
3. Rust 边界清晰：前端统一经 `shared/tauri` 调用。
4. 文档与代码同步：每次 PR 必须更新对应文档。

## 错误处理规范

1. 禁止静默吞错。
2. 用户错误可见，开发错误可追踪。
3. 统一错误类型与日志层。
