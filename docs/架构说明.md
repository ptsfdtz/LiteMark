# 架构说明

## 当前架构（阶段约束）

```text
src/
  App.tsx                # 业务主入口（当前阶段集中开发）
  main.tsx               # React 启动
  index.css              # Tailwind v4 + shadcn token
  components/
    ui/                  # 基础复用组件
  hooks/
    *.ts                 # 自定义 hooks
  lib/
    utils.ts             # 通用工具
  types/
    *.d.ts               # 类型声明集中目录
src-tauri/
  src/lib.rs             # Tauri app 启动
  tauri.conf.json        # 默认窗口与构建配置
```

## 当前阶段不采用的结构

```text
features/*
entities/*
shared/*
```

说明：
1. 上述业务分层结构会在“功能稳定后”再启用。
2. 现阶段避免过度设计和过早拆分。

## 设计原则（当前阶段）

1. 业务先跑通：优先实现功能闭环。
2. 基础与业务分离：`components/ui` 仅放基础组件。
3. 类型集中：类型统一放 `src/types/*.d.ts`。
4. 组件封装按复用触发：不复用的业务片段保留在 `App.tsx`。
5. 命名简洁：避免过长组件名和模块名。
6. 文档同步：结构规则变化必须同步更新文档。

## 错误处理规范

1. 禁止静默吞错。
2. 用户错误可见，开发错误可追踪。
3. 统一错误类型与日志层。
